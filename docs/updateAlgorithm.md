# Предварительный алгоритм обновления модулей управления MESH

## Список необходимого оборудования для проверочного запуска:

1.  ноутбук с выходом в интернет (в идеале 2)
2.  гейта для запуска на разных частотах
3.  wifi роутер на 4 порта для подключения 3 гейтов и ноутбука
4.  lan-кабеля для подключения гейтов к роутеру
5.  сетевые удлинители
6.  источник питания для гейтов
7.  Mesh снифферы (в идеале 3 шт. на каждый канал) - в текущей реализации подключаются по usb к ноутбуку, для мониторинга трафика между гейтами и МУ
8.  mqtt broker (mosquitto)
9.  Mqttx
10.  MLPD bootloader
11.  MLPD assist
12.  Актуальная прошивка (hex файл) для МУ

## Алгоритм обновления МУ на уровне mqtt из hex файла:

1.  Разбить hex файл на строки по символам переноса строки
2.  Каждую строку разбить по 2 символа (1 байт) и преобразовать в массив байт
3.  Удалить строки в которых байт № 3 (при нумерации с 0) не равен 0x00 (4 байт по порядку в hex файле)
4.  Удалить последнюю строку
5.  Собрать из получившегося массива строк данные, записываемые в память МУ: из каждой строки взять байты с 4 по предпоследний и объединить их в один массив байт
6.  Вычислить контрольную сумму от получившегося массива байт с помощью алгоритма CRC16 (Prev = 0xFFFF)
7.  Построчно публиковать топики в mqtt в формате: "{projectName}/{gateName}/all/all/boot" с полезной нагрузкой в виде JSON объекта, содержащего поле "data": массив байт, полученный на шаге 5 (Пример полезной нагрузки: {"Data":[16,16,0,0,224,43,0,32,205,81,0,8,5,82,0,8,9,82,0,8,205]})
8.  По завершении публикации всех строк, опубликовать топик "{projectName}/{gateName}/all/all/boot" запускающий процесс обновления с полезной нагрузкой в виде JSON объекта, содержащего поле "data" состоящего из массива байт: [6,0,0,1,{LEN_L},{LEN_H},{CRC_L},{CRC_H},1,1,{SUM}], где

-   LEN_L и LEN_H - младший и старший байт от длины массива байт, полученного на шаге 5
-   CRC_L и CRC_H - младший и старший байт от контрольной суммы, полученной на шаге 6
-   SUM - 256 - сумма всех байт в сообщении по модулю 256

    ```ecmascript
    const msg = [payload.length, 0x00, 0x00, 0x01, ...Array.from(payload)]
    const sum = 256 - (msg.reduce((acc, cur) => acc + cur) % 256)
    ```


Алгоритм общего обновления:

1.  Отключить все гейты от питания на сутки
2.  Оставить включенными только 3 гейта на разных частотных каналах
3.  При получении отклика от МУ с гейт0 (здесь и далее обозначение гейт1, означает гейт на канале 1, гейт2 - на канале2 ) и наличии версии больше 4.4 отправлять на гейт0 топик для перевода этого МУ на 2 канал
4.  При появлении отклика от МУ с гейт0 без версии отправлять их на канал 1, до 3 шт
5.  Когда на первом канале будет 3 МУ начинать процесс вертикальной прошивки с гейта1
6.  После прошивки подождать 3,5 минуты для выстраивания обновленных контроллеров и старых обратно в сеть
7.  МУ с обновленной прошивкой перевести на канал 2
